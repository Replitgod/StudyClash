# Mastery-first feature gate

StudyJoust's differentiator is depth of learning, not feature count. Before adding a new feature, screen it against these questions; if the answer to all three is no, don't build it yet:

1. Does it close a loop back to practice, retention, or a mistake the student made (not just a passive view)?
2. Does it make an existing weak-topic/mastery signal more accurate or more actionable?
3. If removed, would a student's actual exam performance be worse?

When in doubt, prefer strengthening an existing loop (Mistake DNA, weak-topic rematch, mastery map, VYRA coaching) over adding a new surface. If a feature is justified but doesn't map to these, say so explicitly and let the user decide — don't silently build it.
